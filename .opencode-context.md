## Stack
- **Runtime:** Node.js v24.16.0, pnpm 11.10.0
- **Monorepo:** pnpm workspaces (4 packages: api, web, shared, ui)
- **Frontend:** React 19 + Vite 6 + Tailwind CSS 4 + React Router 7
- **Backend:** Express.js 5 + Prisma ORM + SQLite (dev) / PostgreSQL (prod)
- **Auth:** JWT 15m access + 7d refresh, bcryptjs
- **Testing:** Vitest (11 backend + 2 frontend tests)

## Key Architecture Decisions
- String enums (SQLite doesn't support native enums; Zod validates)
- Prisma model `FAQ` → `prisma.fAQ` in code (camelCase)
- pnpm build config: `onlyBuiltDependencies` in `pnpm-workspace.yaml`
- JWT stored in localStorage; `authenticate` + `authorize` middleware on API
- All routes under `/api/v1/`

## Bugs Fixed This Session
1. **CSS not loading** — Tailwind v4 requires `@tailwindcss/vite` plugin in `vite.config.ts`. Also added Vite alias for `@demp/ui` to resolve CSS imports.
2. **Duplicate tailwind import** — `packages/ui/src/styles/globals.css` had `@import "tailwindcss"` which was redundant (already in `apps/web/src/styles.css`). Removed.
3. **`/profile` route in wrong group** — was under admin-only `ProtectedRoute`, moved to all-authenticated group.
4. **Register page white screen** — Added ErrorBoundary component to catch rendering errors in lazy-loaded pages. Instead of blank white, users now see a user-friendly error message with a reload button.

## API Routes
- `/auth/*` — register, login, refresh, me, profile, password
- `/events/*` — CRUD, clone, bulk-status, registrations list
- `/events/:id/register` — register/deregister
- `/me/registrations` — my registrations
- `/me/notifications` — list, read, read-all
- `/rules/*` — CRUD
- `/faq/*` — CRUD, search?q=
- `/admin/*` — stats, recent-activity
